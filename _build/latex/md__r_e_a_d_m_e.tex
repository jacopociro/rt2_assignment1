Implements the action service (Mov.\+action) that, when called, requests a position , runs the algorithm to control the robot and return a boolean true value as response. The algorithm reads from /odom the current position and orientation and, knowing the goal, calculates the linear and angular speed to send on the /cmd\+\_\+vel topic. All of this is done with a state machine that has three states. \hypertarget{md__r_e_a_d_m_e_autotoc_md3}{}\doxysubsection{position\+\_\+service.\+cpp}\label{md__r_e_a_d_m_e_autotoc_md3}
Implements a service (Random\+Position.\+srv) that, given a range, returns a random position for the robot to reach. \hypertarget{md__r_e_a_d_m_e_autotoc_md4}{}\doxysubsection{state\+\_\+machine.\+cpp}\label{md__r_e_a_d_m_e_autotoc_md4}
This is the node that calls both the services in go\+\_\+to\+\_\+point.\+py and \mbox{\hyperlink{position__service_8cpp}{position\+\_\+service.\+cpp}} and the service that is called in user\+\_\+interface.\+py. This node waits for user\+\_\+interface.\+py to tell it to call the other two services and start the robot.\hypertarget{md__r_e_a_d_m_e_autotoc_md5}{}\doxysection{G\+A\+Z\+E\+B\+O S\+I\+M\+U\+L\+A\+T\+I\+ON}\label{md__r_e_a_d_m_e_autotoc_md5}
